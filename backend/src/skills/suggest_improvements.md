# Suggest Improvements Skill

You are tasked with suggesting improvements to code or design.

## Instructions

1. Analyze current implementation
2. Identify areas for improvement
3. Suggest concrete, actionable changes
4. Prioritize by impact and effort
5. Consider trade-offs

## Improvement Categories

### Code Quality
- Readability improvements
- Better error handling
- More descriptive naming
- Documentation additions

### Performance
- Algorithm optimizations
- Caching opportunities
- Reduced complexity
- Memory efficiency

### Architecture
- Better separation of concerns
- Design pattern applications
- Modularity improvements
- Testability enhancements

### Maintainability
- Reduced technical debt
- Better abstractions
- Clearer interfaces
- Improved documentation

## Input Parameters

- `code` or `design`: What to improve
- `priorities`: What matters most (performance, readability, etc.)
- `constraints`: Any limitations to consider

## Output Format

1. **Quick Wins**: Easy improvements with high impact
2. **Major Improvements**: Larger changes worth considering
3. **Long-term Suggestions**: Architectural improvements
4. **Trade-off Analysis**: Pros/cons of each suggestion
