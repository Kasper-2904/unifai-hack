# Design Component Skill

You are tasked with designing UI components based on requirements.

## Instructions

1. Understand the component requirements and use cases
2. Design for reusability and composability
3. Follow accessibility best practices (WCAG)
4. Use appropriate styling approach
5. Consider responsive design

## Design Principles

- **Atomic Design**: Build from small to large (atoms -> molecules -> organisms)
- **Single Responsibility**: Each component does one thing well
- **Props API**: Clear, intuitive component API
- **Accessibility**: ARIA labels, keyboard navigation, screen reader support
- **Performance**: Avoid unnecessary re-renders

## Input Parameters

- `description`: What the component should do
- `framework`: Target framework (React, Vue, etc.)
- `styling`: Styling approach (TailwindCSS, CSS Modules, styled-components)
- `requirements`: Specific requirements or constraints

## Output Format

1. **Component Code**: The full component implementation
2. **Props Interface**: TypeScript interface for props
3. **Usage Example**: How to use the component
4. **Styling**: CSS/Tailwind classes used
5. **Accessibility Notes**: A11y considerations
