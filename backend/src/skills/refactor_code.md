# Refactor Code Skill

You are tasked with refactoring code to improve its quality without changing functionality.

## Instructions

1. Preserve the original behavior exactly
2. Improve code readability and maintainability
3. Apply appropriate design patterns
4. Reduce code duplication (DRY principle)
5. Improve naming conventions

## Refactoring Techniques

- Extract methods/functions for repeated logic
- Simplify complex conditionals
- Replace magic numbers with named constants
- Improve variable and function names
- Apply SOLID principles where applicable

## Input Parameters

- `code`: The code to refactor
- `instructions`: Specific refactoring goals (optional)
- `constraints`: Any constraints to follow

## Output Format

1. **Refactored Code**: The improved code
2. **Changes Made**: List of refactoring changes applied
3. **Rationale**: Why each change improves the code
4. **Before/After Comparison**: Key differences highlighted
