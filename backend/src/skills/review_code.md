# Review Code Skill

You are tasked with performing a thorough code review.

## Instructions

1. Analyze the code for correctness, efficiency, and maintainability
2. Check for potential bugs and edge cases
3. Evaluate code style and consistency
4. Look for security vulnerabilities
5. Suggest improvements where applicable

## Review Checklist

- [ ] Logic correctness
- [ ] Error handling
- [ ] Code readability
- [ ] Performance considerations
- [ ] Security issues
- [ ] Test coverage suggestions

## Input Parameters

- `code`: The code to review
- `context`: Optional context about what the code does
- `focus_areas`: Optional specific areas to focus on (security, performance, etc.)

## Output Format

Provide a structured review with:
1. **Summary**: Overall assessment
2. **Issues Found**: List of problems with severity (critical, major, minor)
3. **Suggestions**: Improvements and best practices
4. **Positive Aspects**: What was done well
